# Secrets — управление секретами на NixOS

## Проблема

NixOS-конфиг хранится в git. Секреты (пароли, токены, ключи) нельзя хранить
в открытом виде — даже в приватном репо это плохая практика.

## Решение — agenix

Секреты шифруются через age/SSH ключи и хранятся в репо в зашифрованном виде (`.age` файлы).
Без приватного ключа они бесполезны — репо можно спокойно держать публичным.

При `nixos-rebuild switch` agenix автоматически расшифровывает секреты в `/run/agenix/`.

## Что безопасно хранить в репо

- зашифрованные `.age` файлы — **да**
- публичные SSH ключи — **да**
- весь остальной nix-конфиг — **да**

## Что никогда не должно попасть в репо

- приватные age/SSH ключи
- пароли и токены в открытом виде
- API ключи

## Подключение agenix

```nix
# flake.nix
inputs.agenix.url = "github:ryantm/agenix";

# configuration.nix
imports = [ inputs.agenix.nixosModules.default ];
age.secrets.my-secret.file = ./secrets/my-secret.age;
```

Использование секрета в конфиге:
```nix
passwordFile = config.age.secrets.my-secret.path;
```

## Где хранится приватный ключ

Приватный ключ лежит **только на машине**, никогда не в репо:
- `~/.ssh/id_ed25519` — если используется SSH ключ
- `/etc/age/key` — если используется отдельный age ключ

В репо хранится только публичный ключ (в `secrets.nix` для agenix).

## Перенос на новый ПК

1. Скопировать приватный ключ на зашифрованную флешку (или передать через SSH)
2. На новой машине положить ключ на то же место
3. `git clone` репо
4. `sudo nixos-rebuild switch` — agenix расшифрует всё автоматически
5. Флешку после переноса надёжно стереть

Рекомендуется держать зашифрованный бэкап приватного ключа в password manager
(Bitwarden, 1Password и др.) на случай потери или поломки диска.

## Текущее состояние

agenix ещё не подключён. В конфиге есть `initialPassword = "1234"` —
дефолтный пароль для первого входа, не является реальным секретом
(предполагается смена через `passwd` после первого логина).
При подключении agenix его стоит убрать и заменить на `hashedPasswordFile`.

Тогда при переезде на новый ПК — просто `nixos-rebuild switch` и всё поднимается само, ключи на месте.

---

Обычно используют существующий SSH ключ как мастер-ключ для agenix — тот что уже есть в `~/.ssh/id_ed25519`.

Тогда круг замыкается:
- SSH ключ для GitHub → шифруем через agenix → расшифровывается мастер-ключом
- Мастер-ключ (он же SSH ключ для GitHub) → лежит в `~/.ssh/` → его и перекидываем на флешке

То есть на флешке всё равно один ключ — и он же мастер, и он же для GitHub. Просто всё остальное (другие ключи, токены, пароли) уже не нужно перекидывать.
